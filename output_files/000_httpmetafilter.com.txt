As glaciers slowly flow into the ocean, <a href="http://www.youtube.com/watch?v=vZtpHoCl5kk">the end of the glaciers calve, or break off</a>. <div class="copy post">This February, Twitter released <a href="http://twitter.github.com/bootstrap/">Bootstrap 2</a> a rewrite of their earlier Bootstrap code. It's basically a framework that offers barebones styles and functionality. What's of interest, though, is that it uses almost no semicolons <small>(just 15 in over 1k lines of code)</small>, which are normally used to separate lines of code. Instead, the code relies on <a href="http://inimino.org/~inimino/blog/javascript_semicolons">automatic semicolon insertion</a> (ASI). Unfortunately this code breaks when <a href="http://en.wikipedia.org/wiki/Minification_(programming)">minified</a> using  <a href="http://www.crockford.com/javascript/jsmin.html">JSMin</a>. This was <a href="https://github.com/twitter/bootstrap/issues/3057">reported as an issue on Bootstraps's github page</a> which led to a <a href="http://brendaneich.com/2012/04/the-infernal-semicolon/">heated discussion on the topic of ASI</a>.  <span class="smallcopy">[<a href="/114935/The-infernal-semicolon" target="_self">more inside</a>]</span><br>
